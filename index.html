Spring Boot MVC 三层架构 和 ModelAndView

发表于 2021-06-28 分类于 SpringBoot

三层架构是一个分层式的软件体系架构设计，它可适用于任何一个项目。

MVC是一个设计模式，它是根据项目的具体需求来决定是否适用于该项目。

三层架构将整个项目划分为：表现层（UI）、业务逻辑层（BLL）、数据访问层（DAL）。

MVC 即Model（模型），View（视图），Controller（控制）。

MVC是表现层的架构，MVC的Model实际上是ViewModel，即供View进行展示的数据。 ViewModel不包含业务逻辑，也不包含数据读取。

MVC的Model层是提供数据的，即POJO类。

3层架构指：表现层（显示层） 业务逻辑层 数据访问层（持久化）

系统注解

@Override 用于修饰方法，表示此方法重写了父类方法。

@Deprecated 用于修饰方法，表示此方法已经过时。经常在版本升级后会遇到

@SuppressWarnnings 告诉编译器忽视某类编译警告

常用注解

使用 在类名上的注解

@RestController 返回字符串，作用相当于@ResponseBody加@Controller

@Controller 返回URL地址，声明这个类是一个SpringMVC Controller对象。

@Service 声明一个业务处理类（实现非接口类）

安装elementUI_Admin

发表于 2021-06-05 更新于 2021-06-08

今天安装element ui admin 因为git被墙的原因，一直不成功。后来用cnpm才安装成功。

出现的错误多种多样，但基本都是因为墙引起的。用cnpm基本就没有问题了。

*vue-cli · read ECONNRESET * 这个错误出现的解决办法，以前用的是淘宝的源。

PS：这个方法只是解决了一会儿，后面又不行了。

最终我的解决办法是下载那个vue init webpack这个命令要用到的webpack模板文件到本地放在C:\Users\用户名.vue-templates这个目录下. 然后就可以了。主要是网络访问问题。
