单例模式

发表于 2021-05-16 更新于 2021-05-17 分类于 设计模式

饿汉式

会在装载类的时候就创建实例，如果数据多会占用较多内存，不过它是线程安全的。

懒汉式

在单线程下没问题，如果多线程可能会建立多个对象。所以不是线程安全的。

它是在需要的时候才创建对象实例。

双重检测锁模式，线程安全

通过反射破坏单例模式与防范。

枚举单例

枚举本身也是一个类，只是继承了枚举超类，这个算是最好的单例实现了，用反射也很难破解。枚举类是没有无参构造函数的。
